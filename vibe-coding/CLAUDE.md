## 1. 项目概述

### 1.1 这是什么 APP？

**生活财务小管家** 是一款面向个人用户的桌面记账软件，帮助你在电脑上轻松记录每一笔收入和支出，让你清楚知道"钱花在哪里了"。

### 1.2 为什么需要它？

- 告别糊涂账，养成记账好习惯
- 一眼看清每月收支状况
- 数据完全保存在你自己的电脑上，不用担心隐私泄露
- 操作简单，不需要学习复杂的财务知识

### 1.3 平台支持

- ✅ **Windows**（Windows 10 及以上）
- ✅ **macOS**（macOS 12 及以上）

### 1.4 v1 版本范围

**包含的功能**（v1.0 要做的事）:

- 记录支出和收入（金额、日期、分类、备注）
- 查看交易历史记录
- 按月份和分类筛选交易
- 查看月度收支统计和分类占比图表
- 编辑和删除已有的交易
- 自定义添加分类（预设分类不可删除）

**不包含的功能**（v1.0 不做，以后可以考虑）:

- 云同步和多设备同步
- 多用户/家庭共享账本
- 银行账户/信用卡自动导入
- AI 智能预算和理财建议
- 外币和汇率换算
- 发票扫描和 OCR 识别

## 3. 产品功能规格

### 3.1 核心功能清单

| 编号 | 功能 | 描述 |
|------|------|------|
| F1 | 记录支出 | 添加一笔支出：金额、日期、分类、可选备注 |
| F2 | 记录收入 | 添加一笔收入：金额、日期、分类、可选备注 |
| F3 | 交易列表 | 查看所有交易记录，按日期倒序排列 |
| F4 | 筛选功能 | 按月份、分类、收入/支出类型筛选 |
| F5 | 编辑交易 | 修改已有的交易记录 |
| F6 | 删除交易 | 删除某笔交易（需确认） |
| F7 | 月度统计 | 查看某月的总收入、总支出、净结余 |
| F8 | 分类统计 | 查看各分类的支出/收入金额和占比 |
| F9 | 趋势图表 | 查看近几个月的收支变化趋势 |
| F10 | 分类管理 | 查看、隐藏、添加自定义分类 |

### 3.2 功能详细描述

#### 记账表单

- 日期选择器：默认显示当天日期，可点击选择其他日期
- 金额输入框：支持小数点（最多两位），单位人民币元
- 收入/支出切换：两个大按钮，点击切换
- 分类选择：下拉列表或图标网格，显示分类名和 emoji 图标
- 备注输入框：可选填，最多 200 个字符
- 保存按钮：点击后交易立即保存

#### 交易列表

- 按日期倒序显示（最新的在最上面）
- 每条记录显示：日期、分类图标+名称、金额、备注摘要
- 支出金额用红色显示，收入金额用绿色显示
- 支持按月份快速筛选（本月/上月/全部）
- 支持按分类筛选
- 点击某条记录可编辑或删除

#### 统计仪表盘

- 顶部显示三个卡片：本月收入、本月支出、本月结余
- 饼图：显示各支出分类的占比
- 柱状图：显示近 6 个月的收支趋势

### 3.3 非功能性需求

- 📴 **离线运行**: 所有数据存储在本地，不需要联网
- 🚀 **启动速度**: 从点击图标到显示窗口不超过 3 秒
- 🔒 **数据隐私**: 数据 100% 保存在用户电脑上，不会上传到任何服务器
- 📊 **数据容量**: 支持至少 10,000 条交易记录，不出现卡顿
- 💾 **数据安全**: 每次操作自动保存；应用异常退出不丢失数据

4. 预设分类在数据库中用 `is_builtin = 1` 标记

## 7. 数据模型设计

> 以下数据库设计适用于所有三套方案（因为都使用 SQLite）。

### 7.1 交易表 `transactions`

```sql
CREATE TABLE transactions (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,  -- 唯一编号，自动增长
    type        TEXT    NOT NULL,                    -- 'income'(收入) 或 'expense'(支出)
    amount      REAL    NOT NULL,                    -- 金额（正数，单位：元）
    category_id TEXT    NOT NULL,                    -- 分类英文ID，关联 categories 表
    date        TEXT    NOT NULL,                    -- 交易日期，格式 YYYY-MM-DD
    note        TEXT,                                -- 备注（可选，最多200字）
    created_at  TEXT    NOT NULL,                    -- 创建时间，格式 YYYY-MM-DD HH:MM:SS
    updated_at  TEXT    NOT NULL                     -- 最后修改时间，格式同上
);
```

### 7.2 分类表 `categories`

```sql
CREATE TABLE categories (
    id         TEXT    PRIMARY KEY,                  -- 分类英文ID（如 'food_dining'）
    name_zh    TEXT    NOT NULL,                     -- 分类中文名（如 '餐饮'）
    type       TEXT    NOT NULL,                     -- 'income' 或 'expense'
    icon       TEXT,                                 -- emoji 图标（如 '🍽️'）
    is_builtin INTEGER DEFAULT 0,                   -- 1=内置分类, 0=用户自建
    is_hidden  INTEGER DEFAULT 0,                   -- 1=用户隐藏, 0=正常显示
    sort_order INTEGER DEFAULT 0                    -- 排序序号
);
```

### 7.3 初始数据

应用首次启动时，自动插入 [第 4 节](#4-交易分类体系) 中列出的所有预设分类（12 个支出 + 6 个收入 = 18 个）。后续用户可自行添加或隐藏分类。

## 8. 界面设计原则

### 8.1 整体风格

- **简洁至上**: 不要花里胡哨，让用户一眼就能找到要用的功能
- **温和可信**: 暖色调为主，给人感觉放心（这是管钱的 APP）
- **中文优先**: 所有按钮、标签、提示文字都用简体中文

### 8.2 配色方案

| 用途 | 颜色 | 说明 |
|------|------|------|
| 主色调 | 深蓝色 `#1a73e8` | 按钮、标题等关键元素 |
| 收入 | 绿色 `#34a853` | 收入金额、收入分类标记 |
| 支出 | 红色 `#ea4334` | 支出金额、支出分类标记 |
| 背景 | 白色/浅灰 `#f5f5f5` | 主背景色 |
| 卡片 | 白色 `#ffffff` | 卡片/面板背景 |
| 文字 | 深灰 `#333333` | 正文文字颜色 |

### 8.3 字体

- **macOS**: 苹方 (PingFang SC)
- **Windows**: 微软雅黑 (Microsoft YaHei)
- 代码/数字: 等宽字体 (Menlo / Consolas)
- 最小字号: 14px（确保易于阅读）

### 8.4 布局

- **单窗口应用**: 不需要弹出多个窗口
- **左侧导航**: 三个标签页 —— 📝 记账、📋 账单、📊 统计
- 窗口默认大小: 1000 × 680 像素（可调整）

### 8.5 金额显示格式

- 人民币符号: ¥
- 千分位分隔: 例如 ¥1,234.56
- 支出前面加负号或以红色显示: `-¥1,234.56`
- 金额输入时自动校验，只允许数字和一个小数点

## 10. 术语表

> 以下用最简单的语言解释你在使用 Claude Code 构建 APP 过程中可能遇到的技术术语。

| 术语 | 简单解释 |
|------|---------|
| **前端 (Frontend)** | 你能看到和点击的部分 —— 界面、按钮、输入框。就像餐厅的用餐区。 |
| **后端 (Backend)** | 在幕后处理数据和逻辑的部分。就像餐厅的厨房，你看不到但饭菜是那里做的。 |
| **数据库 (Database)** | 存数据的地方，就像一个超级电子表格。你的每一笔记账都存在这里。 |
| **SQLite** | 一种轻量级数据库，把你的数据存在一个文件里，不需要安装数据库软件。 |
| **框架 (Framework)** | 别人已经写好的"半成品房子"，你只需要在此基础上装修就行，不用从零开始。 |
| **库 (Library)** | 别人写好的工具集，你可以直接拿来用。就像工具箱里的扳手。 |
| **依赖 (Dependency)** | 你的项目需要用到别人的代码库。就像做饭需要食材来源。 |
| **打包 (Package/Build)** | 把代码变成可以直接双击打开的应用。就像把食材做成一道菜。 |
| **Git** | 一个版本管理工具，能记录你每次修改了什么。就像写文章的"撤销"功能，但强大得多。 |
| **Commit** | 在 Git 中保存一次修改记录。就像游戏里的存档点。 |
| **CLI (命令行)** | 黑底白字的窗口，通过敲键盘指令来操作电脑。 |
| **npm / pip** | 用来下载别人写好的代码库的工具。npm 是 JavaScript 的，pip 是 Python 的。 |
| **Node.js** | 让 JavaScript 能在浏览器之外运行的环境。我们的 Electron 和 Tauri 方案都需要它。 |
| **API** | 不同程序之间"对话"的方式。你的应用不需要联网，所以这个概念目前不重要。 |
| **CSV** | 一种最简单的表格文件格式，可以用 Excel 或 WPS 打开。 |
| **Bug** | 程序的错误或问题。就像文章里的错别字。 |
